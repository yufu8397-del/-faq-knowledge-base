# デプロイエラー解決方法

## エラー内容
```
Your service has not been deployed because the GitHub repository is empty. 
Make a commit before retrying.
```

## 原因
GitHubリポジトリが空の状態です。コードがプッシュされていない可能性があります。

---

## 解決方法

### ステップ1: GitHubにプッシュを確認

ターミナルで以下を実行して、GitHubにプッシュされているか確認：

```bash
cd /Users/nakajima-yuusuke/Desktop/Cursor/faq-knowledge-base
git push -u origin main
```

**認証情報の入力：**
- **Username**: `yufu8397-del`
- **Password**: Personal Access Token（以前コピーしたもの）

---

### ステップ2: プッシュが成功したか確認

成功した場合、以下のようなメッセージが表示されます：

```
Enumerating objects: XX, done.
...
To https://github.com/yufu8397-del/faq-knowledge-base.git
 * [new branch]      main -> main
```

---

### ステップ3: GitHubで確認

1. **https://github.com/yufu8397-del/faq-knowledge-base にアクセス**
2. **ファイルが表示されているか確認**
   - `package.json`
   - `server/`
   - `client/`
   などが表示されていれば成功

---

### ステップ4: Renderで再デプロイ

プッシュが成功したら：

1. **Renderのダッシュボードに戻る**
2. **「Manual Deploy」ボタンをクリック**
3. **「Deploy latest commit」を選択**
4. **デプロイが開始されます**

または、自動デプロイが有効になっていれば、数分待つと自動でデプロイが開始されます。

---

## プッシュが失敗する場合

### エラー: "Repository not found"
→ GitHubでリポジトリが作成されているか確認

### エラー: "Authentication failed"
→ Personal Access Tokenが正しいか確認
→ トークンが有効期限内か確認

### エラー: "Permission denied"
→ トークンのスコープに `repo` が含まれているか確認

---

## 完了後

デプロイが成功すると：
- ✅ 「Live」と表示される
- ✅ URL `https://faq-knowledge-base.onrender.com` にアクセスできる
- ✅ アプリが動作する
