# 外注者が使えるようにする手順（超簡単版）

## ステップ1: GitHubにコードをアップロード

### 1-1. GitHubでリポジトリを作成
1. https://github.com にアクセスしてログイン
2. 右上の「+」→「New repository」をクリック
3. リポジトリ名を入力（例: `faq-knowledge-base`）
4. 「Create repository」をクリック

### 1-2. ターミナルでコードをアップロード
以下のコマンドを順番に実行してください：

```bash
cd /Users/nakajima-yuusuke/Desktop/Cursor/faq-knowledge-base

# Gitを初期化
git init

# すべてのファイルを追加
git add .

# コミット
git commit -m "Initial commit"

# GitHubリポジトリを接続（[YOUR-USERNAME]と[REPO-NAME]を実際の値に置き換える）
git remote add origin https://github.com/[YOUR-USERNAME]/[REPO-NAME].git

# アップロード
git branch -M main
git push -u origin main
```

**注意**: GitHubのユーザー名とリポジトリ名を入力してください。

---

## ステップ2: Renderで公開

### 2-1. Renderアカウント作成
1. https://render.com にアクセス
2. 「Get Started for Free」をクリック
3. 「Continue with GitHub」を選択してGitHubアカウントでログイン

### 2-2. サービスを作成
1. ダッシュボードで「New +」→「Web Service」をクリック
2. 先ほど作成したGitHubリポジトリを選択
3. 以下の設定を入力：

   **基本設定:**
   - **Name**: `faq-knowledge-base`（何でもOK）
   - **Region**: `Singapore`（日本に近い）

   **ビルド設定:**
   - **Environment**: `Node`
   - **Build Command**: 
     ```
     npm install && cd client && npm install && npm run build
     ```
   - **Start Command**: 
     ```
     node server/index.js
     ```

   **環境変数:**
   - 「Add Environment Variable」をクリック
   - Key: `NODE_ENV`
   - Value: `production`
   - 「Add」をクリック

4. 「Create Web Service」をクリック

### 2-3. デプロイ完了を待つ
- 数分かかります（5-10分程度）
- 「Live」と表示されれば完了です
- URLが表示されます（例: `https://faq-knowledge-base.onrender.com`）

---

## ステップ3: 外注者に共有

デプロイが完了したら、表示されたURLを外注者に共有するだけです！

例：
```
https://faq-knowledge-base.onrender.com
```

外注者はこのURLをブラウザで開くだけで使えます。

---

## トラブルシューティング

### エラーが出た場合

**「Build failed」と表示される場合:**
- Build Commandが正しいか確認
- ログを確認してエラー内容を確認

**「Application error」と表示される場合:**
- Start Commandが正しいか確認
- 環境変数 `NODE_ENV=production` が設定されているか確認

**データが消える場合:**
- Renderの無料プランではデータは永続化されますが、サービスを削除すると消えます
- 重要なデータは定期的にバックアップを取ることをおすすめします

---

## 無料プランの制限

- 15分間アクセスがないとスリープします
- 最初のアクセス時に起動するまで少し時間がかかります（30秒〜1分）
- 月間750時間まで無料

---

## もっと簡単にしたい場合

もしGitHubを使いたくない場合は、以下の方法もあります：

1. **ローカルネットワークで共有**
   - あなたのPCで `npm run dev` を実行
   - 同じWi-Fiに接続している外注者から `http://[あなたのPCのIPアドレス]:3000` にアクセス

2. **ngrokを使う（一時的な公開）**
   ```bash
   npm install -g ngrok
   npm run dev
   # 別のターミナルで
   ngrok http 3000
   ```
   - 表示されたURLを外注者に共有（無料プランは2時間で期限切れ）
