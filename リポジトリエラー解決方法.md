# リポジトリエラー解決方法

## エラー内容
```
remote: Repository not found.
fatal: repository 'https://github.com/yufu8397-del/faq-knowledge-base.git/' not found
```

## 原因
GitHubでリポジトリが作成されていないか、リモートURLが間違っている可能性があります。

---

## 解決方法

### 方法1: GitHubでリポジトリが作成されているか確認

1. **https://github.com/yufu8397-del にアクセス**
2. **「Repositories」タブを確認**
3. **`faq-knowledge-base` というリポジトリが表示されているか確認**

**もし表示されていない場合：**
- リポジトリが作成されていません
- 再度作成する必要があります

### 方法2: リポジトリを再作成

1. **https://github.com/new にアクセス**
2. **Repository name**: `faq-knowledge-base`
3. **Public / Private**: どちらでもOK
4. **「Add a README file」などのチェックを外す**
5. **「Create repository」をクリック**

### 方法3: リモートURLを確認・修正

ターミナルで以下を実行：

```bash
cd /Users/nakajima-yuusuke/Desktop/Cursor/faq-knowledge-base

# 現在のリモートURLを確認
git remote -v

# もし間違っていたら、削除して再設定
git remote remove origin
git remote add origin https://github.com/yufu8397-del/faq-knowledge-base.git

# 再度プッシュを試行
git push -u origin main
```

## Personal Access Tokenの作成（まだの場合）

1. **https://github.com/settings/tokens にアクセス**
2. **「Generate new token (classic)」をクリック**
3. **Note**: `faq-deploy`
4. **Expiration**: 90 days
5. **Select scopes**: ✅ `repo` にチェック
6. **「Generate token」をクリック**
7. **トークンをコピー**

プッシュ時のパスワードとして、このトークンを使用してください。
