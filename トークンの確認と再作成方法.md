# Personal Access Tokenの確認と再作成方法

## 重要：トークンは一度しか表示されません

Personal Access Tokenは、作成時に一度だけ表示されます。その画面を閉じると、二度と見ることができません。

---

## トークンが見つからない場合

トークンを見失った場合は、**再作成する必要があります**。

---

## トークンの再作成手順

### ステップ1: トークン作成ページにアクセス

1. **https://github.com/settings/tokens にアクセス**
   - または、GitHubの右上のアイコン → 「Settings」→ 左メニューの「Developer settings」→ 「Personal access tokens」→ 「Tokens (classic)」

### ステップ2: 新しいトークンを生成

1. **「Generate new token」をクリック**
2. **「Generate new token (classic)」をクリック**

### ステップ3: 設定を入力

1. **Note（メモ）**: 
   ```
   faq-deploy-2
   ```
   など、適当な名前を入力（既存のトークンと区別するため）

2. **Expiration（有効期限）**: 
   - 適当な期間を選択（例: **90 days**）
   - または「No expiration」を選択

3. **Select scopes（スコープ）**: 
   - ✅ **`repo`** にチェックを入れる
     - これでリポジトリへの読み書きが可能になります

4. **ページの一番下の「Generate token」ボタンをクリック**

### ステップ4: トークンをコピー

- **表示されたトークンをすぐにコピーしてください**
- ⚠️ **この画面を閉じると二度と見れません！**
- 例: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

---

## 既存のトークンを確認する方法

既存のトークンは、セキュリティ上の理由で**確認できません**。

ただし、以下の方法で確認できます：

1. **https://github.com/settings/tokens にアクセス**
2. **作成済みのトークンが一覧表示されます**
3. **ただし、トークンの値自体は表示されません**（セキュリティのため）
4. **「Revoke」ボタンで削除できます**

---

## 推奨：新しいトークンを作成

既存のトークンが見つからない場合は、新しいトークンを作成することをおすすめします。

1. **既存のトークンを削除**（必要に応じて）
2. **新しいトークンを作成**
3. **新しいトークンを使用**
