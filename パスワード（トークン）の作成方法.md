# パスワード（Personal Access Token）の作成方法

## 重要：GitHubのパスワードではなく、トークンを使います

GitHubへのプッシュには、通常のパスワードではなく、**Personal Access Token**が必要です。

---

## トークンの作成手順（2分）

### ステップ1: トークン作成ページにアクセス

1. **https://github.com/settings/tokens にアクセス**
   - または、GitHubの右上のアイコン → 「Settings」→ 左メニューの「Developer settings」→ 「Personal access tokens」→ 「Tokens (classic)」

### ステップ2: トークンを生成

1. **「Generate new token」をクリック**
2. **「Generate new token (classic)」をクリック**

### ステップ3: 設定を入力

1. **Note（メモ）**: 
   ```
   faq-deploy
   ```
   など、適当な名前を入力

2. **Expiration（有効期限）**: 
   - 適当な期間を選択（例: **90 days** または **No expiration**）

3. **Select scopes（スコープ）**: 
   - ✅ **`repo`** にチェックを入れる
     - これでリポジトリへの読み書きが可能になります

4. **ページの一番下の「Generate token」ボタンをクリック**

### ステップ4: トークンをコピー

- **表示されたトークンをすぐにコピーしてください**
- ⚠️ **この画面を閉じると二度と見れません！**
- 例: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

2. **Password**: 
   ```
   [コピーしたトークンを貼り付け]
   ```
   （例: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）

---

## 注意事項

- トークンはパスワードのように見えない文字列です（例: `ghp_...`）
- トークンは秘密情報です。他人に共有しないでください
- トークンは一度しか表示されません。失くしたら再作成が必要です
