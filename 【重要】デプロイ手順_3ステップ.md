## ステップ1: GitHubでリポジトリを作成（1分）

1. **https://github.com にアクセスしてログイン**
2. **右上の「+」→「New repository」をクリック**
3. **リポジトリ名を入力**: `faq-knowledge-base`
4. **「Create repository」をクリック**
   - 説明やREADMEは追加しなくてOK
   - プライベート/パブリックはどちらでもOK

## ステップ2: GitHubにコードをプッシュ（30秒）

**ターミナルで以下を実行：**

```bash
cd /Users/nakajima-yuusuke/Desktop/Cursor/faq-knowledge-base

# [あなたのGitHubユーザー名]を実際の値に置き換える
git remote add origin https://github.com/[あなたのGitHubユーザー名]/faq-knowledge-base.git
git branch -M main
git push -u origin main
```

**GitHubユーザー名の確認方法:**
- GitHubの右上のアイコンをクリック
- ユーザー名が表示されます（例: `yufu8397-del`）

**例（ユーザー名が `yufu8397-del` の場合）:**
```bash
git remote add origin https://github.com/yufu8397-del/faq-knowledge-base.git
git branch -M main
git push -u origin main
```

## ステップ3: Renderで公開（5分）

1. **https://render.com にアクセス**
2. **「Get Started for Free」をクリック**
3. **「Continue with GitHub」をクリック**
   - GitHubアカウントでログイン
4. **ダッシュボードで「New +」→「Web Service」をクリック**
5. **リポジトリを選択**: `faq-knowledge-base`
6. **設定を入力:**

   **基本設定:**
   - **Name**: `faq-knowledge-base`（自動入力されます）

   **ビルド設定:**
   - **Build Command**: 
     ```
     npm install && cd client && npm install && npm run build
     ```
   - **Start Command**: 
     ```
     node server/index.js
     ```

   **環境変数:**
   - 「Add Environment Variable」をクリック
   - **Key**: `NODE_ENV`
   - **Value**: `production`
   - 「Add」をクリック

7. **「Create Web Service」をクリック**
8. **5-10分待つ**（ビルド中）
9. **「Live」と表示されれば完了！**

## トラブルシューティング

### ステップ2でエラーが出る場合
- GitHubでリポジトリを作成したか確認
- リポジトリ名が `faq-knowledge-base` か確認
- ユーザー名が正しいか確認

### ステップ3でビルドが失敗する場合
- Build Commandが正しいか確認（コピー&ペースト推奨）
- ログを確認してエラー内容を確認

### アプリが起動しない場合
- Start Commandが正しいか確認
- 環境変数 `NODE_ENV=production` が設定されているか確認
